BlocksKit by zwaldowski

The Objective-C block utilities you always wish you had.

Blocks in C and Objective-C are downright magical. They make coding easier and potentially quicker, not to mention faster on the front end with multithreading and Grand Central Dispatch. BlocksKit hopes to facilitate this kind of programming by removing some of the annoying - and, in some cases, impeding - limits on coding with blocks.

BlocksKit includes: basic block calling; key value observing extensions; filtering and enumeration for mutable and immutable collections; NSInvocation from a block; block-based timers, controls, and gesture recognizers; block-based alert views and action sheets; and block-backed delegation alternatives to NSURLConnection, NSCache, MFMailComposeViewController, MFMessageComposeViewController, and UIWebView.

Dependencies

BlocksKit is a framework and static library for Mac OS X 10.6+ and iOS 4.0+. Use of BlocksKit on iOS requires linking against Core Graphics, MessageUI, and UIKit.

Install

BlocksKit can be added to a project using CocoaPods.

Framework

  1. Download a release of BlocksKit.
  2. Move BlocksKit.framework to your project's folder. Drag it from there into your project.
  3. Add BlocksKit.framework to "Link Binary With Libraries" in your app's target. Make sure your app is linked with CoreGraphics, Foundation, MessageUI, and UIKit.
  4. In the build settings of your target or project, change "Other Linker Flags" to -ObjC -all_load.
  5. #import in your project's prefix header.
  6. Make amazing software.

Library

  1. Download a release of BlocksKit.
  2. Move libBlocksKit.a and Headers to your project's folder, preferably a subfolder like "Vendor".
  3. In "Build Phases", Drag libBlocksKit.a into your target's "Link Binary With Libraries" build phase. Make sure your app is linked with CoreGraphics, Foundation, MessageUI, and UIKit.
  4. In the build settings of your target or project, change "Other Linker Flags" to -ObjC -all_load.
  5. Add an entry to "Header Search Paths" with the relative path to BlocksKit's headers, like $(SRCROOT)/Vendor/Headers.
  6. #import in your project's prefix header.
